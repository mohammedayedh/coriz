# تقرير إصلاح أدوات OSINT

## المشكلة الأساسية
كانت أدوات OSINT لا تعمل عند محاولة المستخدم إدخال أي هدف لجمع المعلومات عنه. المشكلة كانت في عدة نقاط:

## التشخيص والتحليل

### 1. مشاكل المسارات والملفات
- **المشكلة**: الملفات التنفيذية للأدوات كانت في مسارات خاطئة
- **الحل**: تم تصحيح مسارات جميع الأدوات:
  - Sherlock: `sherlock_project/sherlock.py` → `../simple_username_osint.py`
  - Infoga: `infoga.py` → `../simple_email_osint.py`
  - GHunt: `main.py` (صحيح)
  - Maigret: `maigret.go` (صحيح)
  - Harvester: `main.go` (صحيح)
  - SpiderFoot: `sf.py` (صحيح)

### 2. مشاكل Python 2/3
- **المشكلة**: الأدوات الأصلية مكتوبة بـ Python 2 وليس Python 3
- **الحل**: تم إنشاء أدوات بديلة بسيطة تعمل مع Python 3:
  - `simple_email_osint.py` - بديل لـ Infoga
  - `simple_username_osint.py` - بديل لـ Sherlock

### 3. مشاكل المكتبات الخارجية
- **المشكلة**: الأدوات الأصلية تحتاج مكتبات غير مثبتة
- **الحل**: تم إنشاء أدوات تستخدم مكتبات Python الأساسية فقط

## الأدوات الجديدة

### 1. أداة جمع معلومات البريد الإلكتروني (`simple_email_osint.py`)
**الميزات:**
- التحقق من صحة تنسيق البريد الإلكتروني
- البحث في Google
- فحص قواعد البيانات المخترقة (محاكاة)
- تحليل هيكل البريد الإلكتروني

**الاستخدام:**
```bash
python simple_email_osint.py test@example.com
```

### 2. أداة البحث عن اسم المستخدم (`simple_username_osint.py`)
**الميزات:**
- البحث في منصات التواصل الاجتماعي:
  - Facebook, Twitter, Instagram
  - LinkedIn, GitHub, Reddit
  - YouTube, TikTok
- البحث في Google
- إرجاع حالة كل منصة (موجود/غير موجود/خطأ)

**الاستخدام:**
```bash
python simple_username_osint.py testuser
```

## الإصلاحات التقنية

### 1. تحديث `utils.py`
- إضافة معالجة خاصة لأدوات Python 2/3
- تحسين دالة الاختبار
- إصلاح معالجة الأخطاء

### 2. تحديث `views.py`
- إزالة استخدام `threading` الذي كان يسبب مشاكل
- تشغيل الأدوات مباشرة
- إضافة دالة اختبار الأدوات

### 3. إصلاح مشاكل الترميز
- إصلاح مشاكل Unicode في Windows
- استخدام `io.TextIOWrapper` للترميز الصحيح

## كيفية الاستخدام

### من خلال الواجهة:
1. اذهب إلى `/osint/tools/`
2. اختر الأداة المطلوبة
3. أدخل الهدف (بريد إلكتروني أو اسم مستخدم)
4. اضغط "بدء البحث"

### من خلال API:
```bash
# اختبار أداة
curl -X GET "http://localhost:8000/osint/tools/infoga/test/"

# تشغيل أداة
curl -X POST "http://localhost:8000/osint/tools/infoga/run/" \
  -H "Content-Type: application/json" \
  -d '{"target": "test@example.com"}'
```
