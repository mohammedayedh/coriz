## كيفية المساهمة

### 1. Fork المستودع
انقر على زر "Fork" في أعلى الصفحة

### 2. استنسخ Fork الخاص بك
```bash
git clone https://github.com/YOUR_USERNAME/coriza-osint.git
cd coriza-osint
```

### 3. أنشئ فرع جديد
```bash
git checkout -b feature/your-feature-name
```

### 4. قم بالتغييرات
- اكتب كود نظيف ومفهوم
- أضف تعليقات عند الحاجة
- اتبع معايير الكود

### 5. اختبر التغييرات
```bash
python manage.py test
```

### 6. Commit التغييرات
```bash
git add .
git commit -m "Add: وصف واضح للتغيير"
```

### 7. Push للفرع
```bash
git push origin feature/your-feature-name
```

### 8. افتح Pull Request
اذهب إلى المستودع الأصلي وافتح Pull Request

## معايير الكود

### Python
- اتبع [PEP 8](https://pep8.org/)
- استخدم أسماء متغيرات واضحة
- اكتب docstrings للدوال والكلاسات
- حافظ على الدوال قصيرة ومركزة

```python
def calculate_confidence_score(data: dict) -> float:
    """
    حساب درجة الثقة بناءً على البيانات المدخلة.
    
    Args:
        data: قاموس يحتوي على بيانات النتيجة
        
    Returns:
        float: درجة الثقة بين 0.0 و 1.0
    """
    # الكود هنا
    pass
```

### HTML/CSS
- استخدم مسافات بادئة متسقة (4 spaces)
- اكتب أسماء classes واضحة
- تجنب inline styles
- استخدم RTL للعربية

### JavaScript
- استخدم ES6+ features
- اكتب تعليقات للدوال المعقدة
- تجنب global variables
- استخدم `const` و `let` بدلاً من `var`

## عملية Pull Request

### قبل إرسال PR

- ✅ تأكد من عمل الكود بدون أخطاء
- ✅ اختبر على متصفحات مختلفة
- ✅ حدّث التوثيق إذا لزم الأمر
- ✅ تأكد من عدم وجود conflicts

### عنوان PR

استخدم بادئات واضحة:
- `Add:` لإضافة ميزة جديدة
- `Fix:` لإصلاح خطأ
- `Update:` لتحديث موجود
- `Refactor:` لإعادة هيكلة الكود
- `Docs:` لتحديث التوثيق

مثال: `Add: Email OSINT tool with Gravatar support`

### وصف PR

اشرح:
- ما الذي تم تغييره؟
- لماذا تم التغيير؟
- كيف تم اختباره؟
- هل هناك screenshots؟

## الإبلاغ عن الأخطاء

### قبل الإبلاغ
- ابحث في Issues الموجودة
- تأكد من أنه خطأ فعلي
- جرب على آخر إصدار

### معلومات مطلوبة
- وصف واضح للمشكلة
- خطوات إعادة إنتاج الخطأ
- السلوك المتوقع vs الفعلي
- Screenshots إن أمكن
- معلومات البيئة (OS, Python version, etc.)

### قالب Issue

```markdown
## وصف المشكلة
وصف واضح ومختصر للمشكلة

## خطوات إعادة الإنتاج
1. اذهب إلى '...'
2. انقر على '...'
3. شاهد الخطأ

## السلوك المتوقع
ما كان يجب أن يحدث

## Screenshots
إن وجدت

## البيئة
- OS: [e.g. macOS 12.0]
- Python: [e.g. 3.10.5]
- Django: [e.g. 5.2]
- Browser: [e.g. Chrome 120]
```
